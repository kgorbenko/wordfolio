# Wordfolio API Endpoint Test File Refactor

## Overview

This roadmap restructures endpoint integration tests in `Wordfolio.Api/Wordfolio.Api.Tests` to improve maintainability and discoverability for backend contributors. The goal is to replace large area-level files with behavior-specific test files grouped by endpoint area directories, while preserving the exact test coverage and behavior.

The refactor is structural only. Endpoint test logic, scenarios, and assertions must remain equivalent before and after the move. Every area runs in two iterations: first refactor + full verification, then explicit coverage parity verification against the pre-refactor state.

## Specification

### Goals & Success Criteria

- Endpoint tests are grouped under area directories inside `Wordfolio.Api/Wordfolio.Api.Tests/`.
- Each endpoint behavior/route variant is in its own file named `<EndpointBehavior>Tests.fs`.
- No endpoint test file name contains `Async`.
- Namespaces match physical locations (for example `Wordfolio.Api.Tests.Entries` for files in `Wordfolio.Api/Wordfolio.Api.Tests/Entries/`).
- `Wordfolio.Api/Wordfolio.Api.Tests/Wordfolio.Api.Tests.fsproj` keeps explicit ordered compile includes: shared test infrastructure files first, then infrastructure tests, then endpoint test files grouped by area.
- For every refactored area, the same tests exist after refactor (validated in verification iteration via generated inventories and runtime discovery).
- `dotnet fantomas`, `dotnet build`, and `dotnet test` pass for every Refactor step.

### Architecture / Design Decisions

- Pattern to follow: `Wordfolio.Api/Wordfolio.Api.Tests/AGENTS.md` (file organization, fixture usage, seed/assertion rules, endpoint coverage expectations).
- Pattern to follow: existing endpoint test body style from each legacy file being split (keep fixture reset, seeding flow, invocation style, and assertions self-contained in each new file).
- Keep one test class per new file, with constructor-injected `WordfolioIdentityTestFixture` and `IClassFixture<WordfolioIdentityTestFixture>`.
- Keep every new endpoint test file fully self-contained: setup, seed, invoke, assert in the same file.
- Keep test method display names unchanged unless required for compilation correctness.
- Keep compile includes explicit (no wildcard/glob compile includes) and grouped in this order:
  - `TestTokenGenerator.fs`
  - `WebApplicationFactory.fs`
  - infrastructure tests (`Infrastructure/...`)
  - endpoint area files in this order: `Auth`, `Collections`, `CollectionsHierarchy`, `Vocabularies`, `Entries`, `Drafts`, `Dictionary`
  - `StatusTests.fs`
  - `OpenApiTests.fs`

### Coverage Verification Design

- Do not hardcode or maintain test-name inventories in this spec.
- Coverage inventories are generated by the agent during each VerifyCoverage step.
- Source-level parity is checked by extracting test method names from:
  - pre-refactor file at `HEAD^` (via `git show`)
  - current refactored area directory
- Runtime parity is checked from `dotnet test --list-tests` output, filtered by the area namespace prefix.

## Agent Instructions

- Do not change production/API/domain/data-access code; this roadmap only refactors `Wordfolio.Api.Tests` plus roadmap docs artifacts.
- Do not add or update NuGet packages or other dependencies.
- Do not use wildcard compile includes in `Wordfolio.Api.Tests.fsproj`; keep includes explicit and ordered.
- Do not retain legacy monolithic endpoint test files after their area is fully split and included via new files.
- Do not include `Async` in endpoint test file names.
- Commit exactly once per completed checkbox step.

## Execution Protocol

**Refactor:**
Carry out the structural split for the area in the step scope. Create behavior-specific files in the target area directory, move tests without changing intent, update namespaces to match physical paths, update explicit compile includes, and remove the legacy area file from compile includes (and from disk when fully migrated). Then run all Verification Commands except `--list-tests`. Fix failures and re-run. Commit one change set for this step with a message describing the area refactor and purpose.

**VerifyCoverage:**
Verify parity for the same area completed in the preceding Refactor step.

1. Create `docs/wordfolio-api-tests-endpoint-refactor/artifacts/` if missing.
2. Set three area-specific values for this step: `LEGACY_FILE`, `AREA_DIR`, `AREA_NAMESPACE_PREFIX`.
3. Generate source inventories:
   - `git show HEAD^:"$LEGACY_FILE" | rg -o 'member\s+_\.``[^`]+``' | rg -o '``[^`]+``' | sort > "docs/wordfolio-api-tests-endpoint-refactor/artifacts/<area>-before.txt"`
   - `rg -o 'member\s+_\.``[^`]+``' "$AREA_DIR" | rg -o '``[^`]+``' | sort > "docs/wordfolio-api-tests-endpoint-refactor/artifacts/<area>-after.txt"`
4. Diff source inventories and require zero differences:
   - `diff -u "docs/wordfolio-api-tests-endpoint-refactor/artifacts/<area>-before.txt" "docs/wordfolio-api-tests-endpoint-refactor/artifacts/<area>-after.txt"`
5. Run test discovery and save output:
   - `dotnet test "Wordfolio.Api/Wordfolio.Api.Tests/Wordfolio.Api.Tests.fsproj" --list-tests > "docs/wordfolio-api-tests-endpoint-refactor/artifacts/<area>-list-tests.txt"`
6. Validate runtime parity for the area by matching discovered tests under `AREA_NAMESPACE_PREFIX` to the generated `<area>-after.txt` inventory (exact multiset match of test display names).
7. If parity fails, fix the refactor and repeat this protocol.
8. Commit one change set for this step, including parity artifacts and any required code fixes.
